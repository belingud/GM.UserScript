## Context

enhanced-reverse-image-search.js 是 Tampermonkey 用户脚本，提供四种图片来源选择方式：选择本地文件、粘贴 URL、点击页面图片、粘贴剪贴板图片。

用户提出新需求：当剪贴板中有图片时，希望直接粘贴到工具中搜索。

## Goals / Non-Goals

**Goals:**
- 实现剪贴板图片粘贴功能
- 保持与现有 UI 交互方式一致
- 图片上传流程复用现有 tmpfiles.org 方案

**Non-Goals:**
- 不支持粘贴板文本或其他内容
- 不支持多图粘贴
- 不改变现有搜索引擎逻辑

## Decisions

1. **使用粘贴事件对话框（paste event dialog）**
   - 实现：弹出模态对话框，用户在其中按 Ctrl+V（桌面）或长按（移动端）触发 `paste` 事件，从 `event.clipboardData` 提取图片 Blob
   - 优点：无需额外权限请求，跨浏览器兼容性更好，用户体验更流畅
   - 备选方案 `navigator.clipboard.read()` 被放弃，原因是需要用户显式授权剪贴板权限，且部分浏览器支持不稳定

2. **新增"粘贴图片"按钮，与现有三个选项并列**
   - 优点：交互一致，用户容易理解
   - 点击后弹出粘贴对话框，引导用户按 Ctrl+V（桌面）或长按（移动端）粘贴图片

3. **粘贴成功后自动标记为已选择**
   - 流程：点击"粘贴图片" → 读取剪贴板 → 上传图床 → 标记选中 → 用户选择搜索引擎

## Risks / Trade-offs

- **浏览器兼容性问题** → 部分旧浏览器可能不支持 Clipboard API，可降级处理
- **权限拒绝** → 用户拒绝剪贴板权限时，显示友好提示
- **剪贴板无图片** → 检测到无图片时，显示提示信息

## Open Questions

- 是否需要支持 Ctrl/Cmd+V 全局快捷键直接触发粘贴？
- 图片过大时是否需要压缩再上传？
